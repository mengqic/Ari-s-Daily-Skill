### Sample 3
**Context:** Providing feedback  
**Tone:** Polite, direct

```
Really appreciate you submitting the patent application. I reviewed the draft and here's some things that stood out for me.


I think it would be more compelling and clear to update the invention title to something like you have for the design brief, 'Visualization of AI-generated within the user interface for an Integrated Development Environment (IDE)'. 
If you need to use acronyms it's best to define the acronym when it's first used in a document. Yes, people might be aware, but it's good practice.
In the unique features section, did your the competitive analysis influence your design direction? Or was it more for validation and comparison after the fact? Only include it if it impacted or influenced the invention.
Lastly, can I see the files (.jpg) you're submitting with the application.
```

### Sample 4
**Context:** Ansering question  
**Tone:** Polite, direct

```
The main reasons why the team (across UX, CX, Dev, and Product) decided to go with a UI label document, which we were in full agreement with, is:


To have a UI label source of truth document, which CX will be responsible for managing and tracking. 
This will place let burden on UX in keeping the Figma file up-to-date to reflect the latest UI labels during the Review phase. 
Once the UI labels are finalized, UX can update the Figma with the final labels one time.

If UX doesn't do a great job of componentizing UI elements, the same UI may have different labels in different user flows, which creates extra confusion for others. 
We can track UI label document can track changes more clearly. 
It will help keep our Figma files cleaner, e.g. comments and annotations are specific to UX and Dev. 

In the future, when we have a label repository (what label to use for a specific use case); managing, searching, and pulling those labels which were used from a document will be easier than from Figma.


Yes, context of seeing the labels and UI together is good, and that is why we include a screenshot of the UI. If more context is needed, a link to the screen in Figma can be included.

Let me know if you still have concerns.
```
